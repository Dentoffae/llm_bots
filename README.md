# LLM Telegram Боты

Два Telegram-бота на Python, подключённых к языковым моделям OpenAI.

| Бот | Папка | Память | Особенности |
|-----|-------|--------|-------------|
| **Dori** | `Dori/` | Нет — каждое сообщение независимо | Простой и быстрый |
| **Nemo** | `Nemo/` | Да — последние 20 сообщений + SQLite | Тезисы диалога, кнопки, форматированные ответы |

---

## Как это работает

### Dori — бот без памяти
Каждое сообщение пользователя обрабатывается независимо. Бот не помнит предыдущих сообщений.

```
Пользователь → сообщение → [system prompt + текущее сообщение] → OpenAI → ответ
```

### Nemo — бот с памятью и базой данных
Бот помнит контекст диалога и накапливает тезисы о пользователе в SQLite.

```
Пользователь → сообщение
                   ↓
        [system prompt]
        [тезисы из БД]        ← SQLite: накопленные факты о пользователе
        [история диалога]     ← последние 20 сообщений в памяти
        [текущее сообщение]
                   ↓
                OpenAI
                   ↓
        {
          "message": "ответ пользователю",   → отправляется в чат
          "theses":  ["тезис 1", "тезис 2"]  → сохраняется в SQLite + лог
        }
```

Тезисы формулируются автоматически в виде:
> *«Пользователь хочет поехать в Армению»*
> *«Пользователь интересуется монастырём Гегард»*

При следующем сообщении все тезисы из БД подгружаются в промпт — бот «помнит» факты о пользователе даже после перезапуска.

## Требования

- Python 3.10+
- Docker и Docker Compose (рекомендуется для запуска)
- Токен Telegram-бота — создать через [@BotFather](https://t.me/BotFather)
- API-ключ OpenAI — получить на [platform.openai.com](https://platform.openai.com/api-keys)

---

## Быстрый старт через Docker (рекомендуется)

### 1. Создать `.env` файлы

```powershell
# Для Dori
copy Dori\.env.example Dori\.env

# Для Nemo
copy Nemo\.env.example Nemo\.env
```

Открыть каждый `.env` и заполнить:

```env
TELEGRAM_TOKEN=123456789:ABCdef...    # токен от @BotFather
OPENAI_API_KEY=sk-...                  # ключ OpenAI
MODEL_NAME=gpt-4o-mini                 # модель (можно изменить)
```

### 2. Запустить оба бота

```powershell
# Из корня LLM_BOTS/
docker compose up -d --build
```

### 3. Запустить только один бот

```powershell
# Только Nemo
cd Nemo
docker compose up -d --build

# Только Dori
cd Dori
docker compose up -d --build
```

### Полезные команды Docker

```powershell
# Посмотреть логи в реальном времени
docker compose logs -f nemo
docker compose logs -f dori

# Остановить всё
docker compose down

# Пересобрать после изменений в коде
docker compose up -d --build
```

## Запуск без Docker (локально)

### 1. Создать виртуальное окружение

```powershell
# Из корня LLM_BOTS/
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# Если PowerShell блокирует запуск скриптов:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 2. Установить зависимости

```powershell
# Сразу для обоих ботов
pip install -r Dori\requirements.txt -r Nemo\requirements.txt
```

### 3. Заполнить `.env` и запустить

```powershell
# Запуск Dori
cd Dori
python bot.py

# Запуск Nemo (в отдельном терминале)
cd Nemo
python bot.py
```

## Команды бота Nemo

| Команда / Кнопка | Действие |
|------------------|----------|
| `/start` | Приветствие, очистка истории диалога |
| `/help` | Справка по командам |
| `📋 Мои тезисы` | Показать все тезисы из базы данных |
| `🗑 Очистить историю` | Очистить историю диалога и тезисы из БД |
| `/theses` | То же, что кнопка «Мои тезисы» |
| `/clear` | То же, что кнопка «Очистить историю» |

| Команда | Действие |
|---------|----------|
| `/start` | Приветствие |

## Настройка через `.env`

### Общие для обоих ботов

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `TELEGRAM_TOKEN` | — | Токен бота от @BotFather |
| `OPENAI_API_KEY` | — | API-ключ OpenAI |
| `MODEL_NAME` | `gpt-4o-mini` | Модель OpenAI (`gpt-4o`, `gpt-4-turbo` и др.) |
| `SYSTEM_PROMPT` | см. `.env.example` | Личность и инструкции бота |

### Только для Nemo

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `MAX_HISTORY` | `20` | Количество сообщений в памяти (в рамках сессии) |
| `MAX_TOKENS` | `3000` | Максимум токенов в ответе модели |
| `TEMPERATURE` | `0.7` | Креативность: `0.0` — точно и сухо, `2.0` — творчески |
| `LOG_LEVEL` | `INFO` | Уровень логов: `DEBUG`, `INFO`, `WARNING`, `ERROR` |
| `DB_PATH` | `nemo.db` | Путь к SQLite-файлу (в Docker задаётся автоматически) |

## База данных (только Nemo)

Nemo хранит тезисы диалога в SQLite-файле `nemo.db`.
Для каждого пользователя создаётся отдельная таблица.

**Схема таблицы** `user_<telegram_id>`:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER | Автоинкремент |
| `thesis` | TEXT | Текст тезиса |
| `created_at` | DATETIME | Дата и время записи |

**В Docker** база данных хранится в именованном volume `nemo-data` и **не теряется** при перезапуске контейнера.

## Логи (только Nemo)

Логи пишутся одновременно в консоль и в файл `Nemo/logs/nemo.log`.

- Ротация: максимум **5 МБ**, хранится **3 резервных копии**
- Уровень в консоли управляется через `LOG_LEVEL` в `.env`
- В файл всегда пишется уровень `DEBUG`

Пример записи в логе:
```
2026-03-27 16:45:01 | INFO     | __main__ | IN  | chat_id=123456789    user=Иван (id=123) | Хочу поехать в Армению
2026-03-27 16:45:03 | INFO     | __main__ | OUT | chat_id=123456789    tokens=(410+95=505)
2026-03-27 16:45:03 | INFO     | __main__ | THESES | chat_id=123456789
  [1] Пользователь хочет поехать в Армению
  [2] Пользователь интересуется достопримечательностями Еревана
```
